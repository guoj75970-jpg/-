## ① 数据下载

### 目标
从虎牙后台下载**前一天（T-1）**的审核数据

### 输入
| 项目 | 说明 |
|------|------|
| 数据日期 | 昨天（例如：今天是7月1日，则下载6月30日数据）|
| 数据来源 | 虎牙审核后台 https://audit.huya.com |

### 操作步骤
1. 登录虎牙审核后台
2. 选择日期范围：**昨天**（例如：2026-06-30）
3. 导出审核明细Excel
4. 下载排队时长数据
5. 下载通道分布数据
6. 保存到本地目录：`D:\HY-日度数据监测\`

### 输出文件
| 文件名 | 说明 |
|--------|------|
| `虎牙审核明细_YYYYMMDD.xlsx` | 审核明细数据 |
| `排队时长_YYYYMMDD.xlsx` | 排队时长数据 |
| `通道分布_YYYYMMDD.xlsx` | 通道分布数据 |

### 验收标准
- [ ] 文件已下载到指定目录
- [ ] 文件大小 > 0
- [ ] 数据日期正确

## ② 异常排查拦截

### 目标
识别并处理**前一天**数据的异常情况

### 输入
- 步骤①下载的数据文件

### 排查项
| 异常类型 | 判断标准 | 处理方式 |
|---------|---------|---------|
| 数据缺失 | 某时段数据量为0 | 联系技术排查 |
| 数据突增/突降 | 较昨日变化>30% | 核实是否为真实业务变化 |
| 排队时长异常 | 平均排队>3分钟 | 预警并分析原因 |
| 违规率异常 | 违规率>15%或<5% | 排查是否有批量误审 |
| 人效异常 | 人均审核<100条 | 排查人员出勤或系统问题 |

### 输出
| 文件名 | 说明 |
|--------|------|
| `异常排查记录_YYYYMMDD.xlsx` | 异常记录 |

### 验收标准
- [ ] 已检查所有异常项
- [ ] 异常已记录
- [ ] 异常已分类（技术/业务/人员）

## ③ 产能折算

### 目标
将**前一天**实际审核量折算为标准产能

### 输入
- 步骤①的审核明细数据

### 折算规则
| 通道类型 | 折算系数 |
|---------|---------|
| 虎牙视频 | 1.0 |
| 虎牙直播回放切片视频 | 0.8 |
| 虎牙AI礼物特效视频 | 0.5 |
| 虎牙一起看视频 | 1.2 |

### 计算公式
```
转化后审核量 = Σ(各通道实际审核量 × 对应折算系数)
```

### 输出
| 指标 | 说明 |
|------|------|
| 实际审核量 | 前一天实际审核条数 |
| 转化后审核量 | 折算后的标准产能 |
| 日均人效 | 转化后审核量 ÷ 工作人数 |

### 验收标准
- [ ] 各通道数据已分类
- [ ] 折算系数已应用
- [ ] 转化后审核量已计算

## ④ KPI监控预警

### 目标
监控**前一天**核心KPI，触发预警

### 输入
- 步骤③的产能数据

### KPI指标
| KPI | 目标值 | 预警阈值 | 严重阈值 |
|-----|--------|---------|---------|
| 日均审核量 | >2500条 | <2000条 | <1500条 |
| 平均排队时长 | <2分钟 | >3分钟 | >5分钟 |
| 平均违规率 | 10-15% | <5%或>20% | <2%或>30% |
| 日均人效 | >150条/人 | <120条/人 | <100条/人 |

### 预警机制
| 预警级别 | 触发条件 | 通知方式 |
|---------|---------|---------|
| 🟢 正常 | 所有KPI达标 | - |
| 🟡 关注 | 1项KPI触发预警 | 企业微信消息 |
| 🟠 警告 | 2项KPI触发预警 | 企业微信+邮件 |
| 🔴 紧急 | 1项KPI触发严重阈值 | 企业微信+邮件+电话 |

### 输出
| 文件名 | 说明 |
|--------|------|
| `KPI监控日报_YYYYMMDD.xlsx` | KPI数据 |
| 预警通知 | 如触发 |

### 验收标准
- [ ] 6项KPI已计算
- [ ] 已判断是否触发预警
- [ ] 预警已发送（如触发）
